# width:auto的表现形式

  • 针对普通文档流的元素，如果容器的宽度需要和父容器的宽度保持一致的话，无需加上width: 100%的属性，可以让自容器利用浏览器原生的流属性布局自适应充满，这个即无宽度原则。
  1. 充分利用可用空间——主要是针对普通文档流的元素，会默认充满整个父容器；
  2. 收缩和包裹——主要是针对绝对定位流，浮动以及inline-block元素或者table元素，表现的是尺寸由内部元素决定；但是绝对定位元素也存在特殊情况，其宽度可以由父元素决定，就是当left/right或者top/bottom同时存在的时候会出现这种情况，如下所示，子元素的宽度是1000px - 20px -20px，这个也被称为格式化宽度；
  3. 收缩到最小——主要是在table元素中的auto表格中；
  4. 超出容器限制——针对设置了white-space:nowrap的元素，内容超出不换行。
  div {
    position: relative;
    width: 1000px;
  }
  div > p {
    position: absolute;
    left: 20px;
    right: 20px;
  }

# box-sizing: border-box的应用

  • 宽度的效果不是作用在传统的content-box上面，即内容盒子上面，而是作用在content-box,padding-box,border-box上面，为了解决宽度自适应的问题，不让容器超出布局范围。

# height: auto应用

  1. 场景：让背景图片充满整个body，要保证父元素有可以计算的固定高度，子元素的100%高度才算有意义的。
  html, body {
    height: 100%;
  }
  .div {
    height: 100%;
  }

  1. 普通文档流下的子元素设置100%高度时不会把父元素的padding计算在内，而绝对定位元素的高度就会计算为和父元素同等高度，即包含padding值。

# min-width/max-width,min-height/max-height的应用

  • 用于规范限制元素的变化布局
  1. 比如不知尺寸的图片
  img {
    max-width: 100%;
    height: auto !important; // 保证图片的宽高比例和原图一致性
  }
1
2
3
4
  1. css的权重： min-width > max-width > !important
  2. max-height应用到收起和展开的动画效果
  .element {
    max-height: 0;
    overflow: hidden;
    transition: max-height .25s;
  }
  .element.active {
    max-height: 666px; /* 一个足够大的最大高度值 */
  }

# 内联元素的特性

  • img是内联元素，默认值是display:inline;但是它由于它是置换元素，所以可以设置宽高，置换元素是指根据标签属性来显示的元素，img,textarea,input,select等都是的。
  1. 内联盒模型——使得其中的元素排成一行展示
  2. 幽灵空白节点——内联元素的所有解析和渲染表现就如同每个行框盒子的前面有一个“空白节点”一样，即内联元素前面有一个宽度为0的空白字符
