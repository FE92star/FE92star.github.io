# 执行机制

# Q1：如何在保证页面流畅的情况下处理海量数据？

  • 业务场景：10w条记录的数据，一次性渲染到页面上，如何处理可以不冻结UI
  • 业务分析：10w条数据量对于浏览器而言是很大的量，直接循环一次性往页面中添加的话，每次循环都会导致页面的回流重绘，从而使页面卡死
  • 优化方案如下：
    1. 减少DOM操作次数
    2. 减少循环时间
  • 具体操作方案如下：
    1. 将海量数据做分批次添加，即将对应添加数据先循环存储到文档碎片中，再进行一次DOM操作添加到文档中去，完成之后进行下一批次的添加
    2. 用requestAnimationFrame保证每次添加都是再下一次浏览器重绘之前，保证浏览器的重绘效率

# Q2：如何使用Promise进行串行或并行操作？

# 串行和并行

  • 串行：上一个promise返回的结果作为下一个promise执行的参数
  • 并行：多个promise异步执行，互不干扰

# Promise串行

  • 应用场景：多重回调嵌套，返回结果作为参数进行传递
  • 写法分析：主要是利用数组方法reduce，上一次执行的结果作为下一次执行的参数，具体写法如下
  // promise的串行
  const runPromiseByQueue = (myPromises) => {
    myPromises.reduce((previousPromise, nextPromise) => previousPromise.then(() => nextPromise()), Promise.resolve())
  }
  const createPromise = (time, id) => () =>
    new Promise(solve =>
      setTimeout(() => {
        console.log("promise", id);
        solve();
      }, time)
    );

  runPromiseByQueue([
    createPromise(3000, 1),
    createPromise(2000, 2),
    createPromise(1000, 3)
  ])

# Promise并行

  • 应用场景：某个接口返回正确结果的前提是前面多个接口有返回
  • 写法分析：利用promise.all方法
  // promise的并行
  const makePromise = (value) => {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        resolve(value)
      }, Math.random() * 1000)
    })
  }
  let promises = [1,2,3,4,5].map((item, index) => {
    return makePromise(item)
  })
  Promise.all(promises)
  .then(() => {
    console.log('done');
  })
  .catch(() => {
    console.log('err');
  })

# Q3：异步解决方案的最佳实现——async/await

  • async/await 是基于promise封装的异步解决方案，async放在function前面表示函数体内有异步操作，await表示需要等待返回结果的异步操作，只有等队列中的await都有返回结果后才会执行then回调
      const promiseFn = () => {
    return new Promise((resolve, reject) => {
      _this.$ajax({
        url: 'http://test.jianbing.com/invest2/account/queryAccount/accountInfo',
        type: "POST",
        success: (r) => {
          _this.value = r.resData
          resolve(_this.value)
        },
        error: (e) => {
          reject(e)
        }
      })
    })
  }
  const promiseFns = (value) => {
    let p = new Promise((resolve, reject) => {
      setTimeout(() => {
        resolve(value)
      }, 2000)
    })
    return p
  }
  const asyncFn = async () => {
    let data = await promiseFn()
    let datas = await promiseFns(data)
    return datas
  }
  asyncFn().then((value) => {
    console.log(value);
  })
  // 最终打印结果是接口的返回值

# Q4：如何用setTimeout模拟setInterval

  • 分析：setInterval表示定时执行任务，setTimeout表示延时执行任务，用setTimeout模拟定时任务，即需要在函数体内再调用函数
      function timer() {
    console.log('time');
    let time = setTimeout(() => {
      timer()
      clearTimeout(time)
    }, 1000)
  }
  timer()
