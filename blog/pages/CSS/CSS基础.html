类属性选择器 描述
[foo^="bar"] 选择foo属性值以bar开头的所有元素
[foo$="bar"] 选择foo属性值以bar结尾的所有元素
[foo*="bar"] 选择foo属性值包含bar字符串的所有元素

# 选择器

  1. 属性选择器 img[alt='github']；
  2. 类属性选择器 p[class~="types"],匹配所有包含类名types的p元素；
  3. 子元素选择器 div > p (子结合符) 表示父元素的子元素，即后代元素，可以匹配一个或多个；
  4. 相邻兄弟元素 h1 + p (兄弟选择符) 表示拥有共同父元素的兄弟元素，可以匹配一个或多个；
  5. 可以多个选择器结合 html > body table + ul——表示选择和table相邻的ul元素，table又是body的后代元素，body是html的子元素；
  6. 伪类选择器 :first-child——选择第一个子元素。

# 定位属性

  • 脱离文档流的布局方式，很常用的布局，但也不能滥用。
  • 对应属性值：static, relative, absolute, fixed, stickey。
    1. static：不具有定位属性，默认值；
    2. relative：相对于元素在文档流定位之前的位置进行定位的；
    3. absolute：相对于最近的具有定位属性的父元素进行定位，当没有定位父元素，即会相对于body定位；
    4. fixed：相对于viewpoint进行定位，即固定定位，常用于导航栏的定位；
    5. stickey：粘性定位。

# CSS👎坑

# 一.固定定位（position: fixed）的坑

  1. 问题描述：固定定位元素在滚动容器里面，滚动容器设置overflow-y: scroll/auto;在IOS系统中，固定定位元素会在页面滚动过程中不能及时保持在固定的位置，会有一个位移过程；
  2. 猜测原因：页面在滚动过程中需要不停的计算该固定定位元素相对于滚动视窗的位置，就会触发重绘，回流的渲染过程，这个过程需要时间，发生肉眼可见的变化；
  3. 解决方式：将滚动容器和固定定位元素作为兄弟元素而不是父子元素进行布局，这样固定元素不再相对于滚动容器布局，而是相对于外面的body或者其他不变父元素进行布局，就不会触发重绘机制
  <!-- 父子层级关系 -->
  <div class="page-scroll">
    <div class="box-fixed"></div>
  </div>
  <!-- 变成兄弟元素 -->
  <div class="page-scroll"></div>
  <div class="box-fixed"></div>

# 二.IOS系统中页面超出滚动不流畅的坑

  • 解决方式：对应滚动容器加上-webkit-scrolling-touch: scroll;

# 三.CSS3动画卡顿，尤其是IOS系统

  • 针对系统不同导致渲染的差异，主要是因为不同系统搭载的内置浏览器的内核不同。
    1. 问题原因：在浏览器中有两个线程来执行页面的渲染工作，主线程和合成线程；主线程负责计算，合成线程负责绘制； 当直接用animation进行动画操作的时候，页面的计算量大大增加，主线程被大量占用，导致页面卡顿；
    2. 解决方式：开始GPU硬件加速提高动画性能——
    3. 给动画元素添加CSS属性：will-change:transform; //提醒浏览器提前准备创建新的图形层；
    4. 用-webkit-transform: translateZ(0);-webkit-t ransform:transition3d(0,0,0) //3d动画开启GPU加速。

# 四.微信客户端图片上增加点击事件会使图片进入预览模式

  • 解决方式：
    1. 增加css属性——pointer-events: none;
    2. VUE中用事件修饰符取消默认事件——@click.prevent;
    3. 对应事件放在外层元素上面即可;
    4. 用原生的取消默认事件方式——e.preventDefault()

# 四.position: stickey的用法总结

  • 定位元素新增属性，粘性定位。
    1. 特点：粘性定位是相对定位和固定定位的结合，应用范围是定位元素在页面滚动中由相对定位变为固定定位的布局需求；
    2. 表现形式：滑动过程中，定位元素在距离其父元素的距离达到stickey的设置距离后，该元素即表现为固定定位；
    3. 使用条件： a、父元素不能overflow:hidden或者overflow:auto属性； b、必须指定top、bottom、left、right4个值之一，否则只会处于相对定位； c、父元素的高度不能低于sticky元素的高度； d、sticky元素仅在其父元素内生效。
