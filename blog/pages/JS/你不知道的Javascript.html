# 你不知道的Javascrpit(11.13——JS编译原理)

# 1. 作用域

  • 作用域在不同的语言设计中都是存在的，它可以被理解为是一种规则，一种对于如何管理变量的内部规则。
  • 分析作用域在JS当中的作用，首先需要理解JS的编译流程，JS是编译型语言，这里涉及到编译型语言和解释型语言的差异：编译型语言是通过编译器将源代码编译成机器可以识别的机器语言再运行，一次编译之后就可以重复运行，而解释型语言则需要每次通过解释器和源代码进行逐行解释和运行，可移植性强，典型的解释型语言是Python
  • 语言编译阶段：
  1. 词法分析/分词——将源代码字符串分解成有意义的代码块，这些代码块被称为词法单元，eg:var a = 2;对应被分解为var,空格,a,=2几个词法单元
  2. 语法分析/解析——将词法单元转换成一个由元素逐级嵌套代表着程序语法结构的树，这个树就被称作抽象语法树——AST
  3. 代码生成——将AST语法转化为可执行代码的过程，这个过程和语言，目标平台相关
  4. 对于JS而言，JS引擎在编译阶段还包含性能优化，通常编译是发生在代码执行前的几微秒，JS引擎通过JIT进行延时编译甚至实施重编译来提升性能。

# 1.1 参与编译的对象

  1. JS引擎——从头到尾负责整个JS的编译以及运行
  2. 编译器——负责语法分析以及代码生成等脏活累活，同引擎合作
  3. 作用域——负责收集并维护由所有声明的标识符（变量）组成的一系列查询，并实施一套非常严格的规则，确定当前执行的代码对这些标识符的访问权限

# 2. 词法作用域

  • 定义是在词法分析阶段的作用域，即词法作用域取决于你在写代码的方式，将对应的变量和块级作用域写在哪里决定，无论函数在哪里调用，或者以什么样的方式调用，只取决于声明时所处的位置决定
  • 编译器在词法分析阶段就可以确定所有标识符被定义的位置，以及如何查找这些标识符。

# 3. 函数的作用域和块级作用域

  • 函数可以创建属于自己的作用域气泡，但是同样一些代码块也可以生成自己的作用域气泡。

# 3.2 立即执行函数表达式(IIFE)

  • 用(fn(){})()这种方式来执行函数，第一个括号内部就是一个函数表达式，一般应用于作为函数的调用并传递参数进去
(
  function fn(global, undefined) {
    return global
  }
)(window)
// 在很多类库的开发过程种可以见过这种IIFE的写法，jquery，sea.js都是这种写法，好处是内部可以通过global替代window，不直接操作window，而undefined作为形参不入实参是防止由于误操作的情况下，将全局的undefined变量污染，因此用这种方式

# 3.3 块作用域

  • 块是代码块，代码块一般是指用{}包裹的代码，在for循环或者if条件判断里面都属于块级作用域的作用范畴，但是var定义的变量在这些代码块种并没有块级作用域的表现，因此ES6新提出的let和const可以隐式的创建一个独立的块级作用域
for(let i=0; i<5; i++) {
  setTimeout(function timer() {
    console.log(i)
  }, i * 1000)
}
// 每次循环的时候都会创建一个独立的块级作用域，保证对变量i的引用，因此每次等循环结束执行setTimeout打印出来的值不一样，而如果用var定义i的时候，i此时已经是全局作用域的变量，因此会打印出相同的值，循环之后i的值变成5
for(var i=0; i<5; i++) {
  (function(j) { // 闭包的作用域链
    setTimeout(function timer() {
      console.log(j)
    }, j* 1000)
  })(i)
}
// 这种解决方式是通过IIFE形成一个独立的块级作用域，保证每次循环之后保存对i的引用，从而实现let定义的功能

# 4. 函数声明和变量在编译阶段会存在提升

  • 函数调用可以在函数声明之前，变量引用可以在变量声明之前，虽然是undefined，而let和const则不存在变量提升的问题，一旦前置引用则直接报错。

# 5. 闭包的定义以及应用

  • 定义：函数具有引用其他作用域变量的能力，并且能使得垃圾回收机制失效。
function bar() {
  var i = 1
  function baz() {
    console.log(i)
  }
  return baz
}
var baz = bar()
baz() // 1——经典闭包效果，bar执行结束之后按照正常逻辑，其作用域是需要被回收的，但是由于baz保留了对bar作用域内部变量的使用，导致bar的作用域一直存在，知道baz函数被调用结束才会被回收

  • 闭包应用在模块管理中，模块模式实现需要两个条件：
  1. 必须包含一个外部的封闭函数，并且该函数至少被调用一次
  2. 封闭函数至少包含一个内部函数，这样的话内部函数会在私有作用域中形成闭包，并且可以访问并修改私有的状态
var foo = (function CoolModule() {
  var something = "cool";
  var another = [1, 2, 3];
  function doSomething() {
    console.log( something );
  }
  function doAnother() {
    console.log( another.join( " ! " ) );
  }
  return {
    doSomething: doSomething,
    doAnother: doAnother
  };
})();
foo.doSomething(); // cool
foo.doAnother(); // 1 ! 2 ! 3

var Modules = (
  function manage() {
    var modules = {}
    function define(name, deeps, impl) {
      for(var i=0; i<deeps.length; i++) {
        deeps[i] = modules[deeps[i]]
      }
      modules[name] = impl.apply(impl, deeps)
    }
    function get(name) {
      return modules[name]
    }
    return {
      define: define,
      get: get
    }
  }
)()
