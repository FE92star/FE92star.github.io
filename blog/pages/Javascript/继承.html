# 《第6章——面向对象的程序设计——6.3继承》

# 继承的定义

  • 继承是oo(面向对象)语言的原则之一（封装，继承，多态），在js中只有实现继承这一种继承方式，并且原型链继承是实现继承的主要方式之一

# 原型链

  • 每一个函数都有一个prototype属性（原型对象），原型对象都包含一个指向构造函数的指针（constructor），每一个用构造函数new出来的实例对象都包含一个指向原型对象的内部指针，大概的形式是：
  obj = new Object()
  obj.__proto__ === Object.prototype
  Object.prototype.constructor === Object

# 一.原型链实现继承

  • 把另外一个构造函数的实例对象赋值给当前函数的原型对象，当前函数的原型就包含指向另一个原型的指针，而另外一个原型又包含指向另外一个构造函数的指针，假如另外一个原型是另一个类型的实例，这种指向关系依然成立，层层递进：
Sub.prototype = new Super()
// 实现Sub继承了Super的方法和属性

# 原型链继承的问题

  • 原型链虽然方便，但是也存在两个主要的问题：
  1. 定义在原型对象中的属性如果是引用类型，就会被所有的实例共享，因此在新原型上修改引用类型的属性值会直接反应在原来的原型对象上面；
  2. 创建子类型的实例时，不能向超类型的构造函数中传递参数；

# 二.构造函数实现继承

  • 运用原生方法call和apply实现继承，主要原理：
 function Super(name) {
   this.name = name
 }
 function Sub() {
   Super.call(this, 'Bob') // 将Super的this指向变到Sub上面，即Sub继承了Super
 }
 let a = new Sub()
 console.log(a.name === 'Bob') // true

# 构造函数实现的问题

  • 所有的方法也必须要在构造函数里面定义，无法实现复用。

# 三.组合继承

  • 即原型链技术和构造函数技术组合到一起实现继承，构造函数定义属性，原型定义方法，是常用的基本上没什么缺陷的继承方式。
