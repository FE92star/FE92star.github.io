# 脚手架自动化生成工具

# yeoman

  • yeoman是一个现代化前端项目的脚手架生成工具，用于生成包含指定框架结构的工程化目录结构，前端自动化的工具。
  • 使用流程：
    1. 全局安装yo模块，此模块是yeoman的主要工具模块；
    2. 选择需要的社区模板进行下载，安装generator-fountain-webapp模块，
    3. 开启命令行输入yo 项目名称就可以开始根据交互命令行输入项目参数，输入完成之后即可开始生成项目；
    4. 通过在项目中安装yeoman-generator，官方的生成器，用于自定义生成自己的脚手架项目，
    5. generator生命周期：initianlizing——获取当前项目状态，获取基本参数等，prompting——向用户展示交互式问题用于收集关键自定义参数，configuring——保存配置相关信息并生成配置文件（名称多为'.'开头的配置文件，例如.editorconfig），default——未匹配任何生命周期方法的非私有方法都是在这里自动执行，writing——依据模板的配置进行新项目结构的文件书写操作，conflicts——处理冲突（内部调用，一般不做处理），install——使用指定的包管理工具进行依赖安装（支持yarn，npm，bower），end——结束动作，如清屏，输出结束信息，say-goodbye等
    6. 项目完成经过测试之后，在项目根目录下执行npm link，即将脚手架安装到npm全局包的安装位置，然后执行yo 自定义模板名称即可快速安装项目。

# commander+inquirer+chalk

  • commander是一个提供了用户命令行输入和参数解析的强大功能，类似于可以识别node -v/-help等用户输入，自定义输出对应的配置版本或者帮助信息，可以链式调用，包含下列选项：
    1. command – 定义命令行指令，后面可跟上一个name，用空格隔开，如 .command( ‘app [name] ‘)(app是全局安装的可执行文件命令)
    2. alias – 定义一个更短的命令行指令 ，如执行命令$ app m 与之是等价的
    3. description – 描述，它会在help里面展示
    4. option – 定义参数。它接受四个参数，在第一个参数中，它可输入短名字 -a和长名字–app ,使用 | 或者,分隔，在命令行里使用时，这两个是等价的，区别是后者可以在程序里通过回调获取到；第二个为描述, 会在 help 信息里展示出来；第三个参数为回调函数，他接收的参数为一个string，有时候我们需要一个命令行创建多个模块，就需要一个回调来处理；第四个参数为默认值
    5. action – 注册一个callback函数,这里需注意目前回调不支持let声明变量
    6. parse – 解析命令行
  • inquirer是一个提供用户输入交互的工具，包含多种功能：input–输入，validate–验证，list–列表选项，confirm–提示，checkbox–复选框等等，最终会获取到用户的交互参数，针对这些输入参数自定义生成个性化的工程项目。
  • chalk是一个用于美化命令行的node工具，即将命令行用不同的颜色渲染输出。 参考链接
