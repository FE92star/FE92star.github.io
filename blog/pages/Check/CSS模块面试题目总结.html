# CSS总结

# Q1：如何理解盒模型？

  • CSS盒模型（Box Model）本质上是一个盒子，封装周围的HTML元素，它包括：边距，边框，填充和盒子实际内容；
  • 盒模型包含两种不同模型：
    1. W3C盒子模型（标准盒模型）——CSS设置的width和height属性是content内容的宽高，盒子的总宽高=width/height+padding+border+margin；
    2. IE盒子模型（怪异盒模型）——CSS设置的width属性和height属性是内容，内边距和边框的总和，盒子的总宽高=width/height+margin=内容+padding+border+margin；
  • 指定盒子模型种类的属性：box-sizing: content-box/border-box；分别用于设置标准盒模型和怪异盒模型。

# Q2：如何理解BFC？

  • BFC——块级格式化上下文，即web页面可视化css渲染的一部分，块盒子的布局过程发生的区域，也是浮动元素和其他元素交互的区域；类似形成了一个独立的容器，容器内部的布局和外部不相干；
  • 如何创建格式化上下文：根元素，浮动元素，绝对定位元素，行内块元素，overflow不是visible的元素，弹性元素flex，网格元素grid
  • BFC的作用：
    1. 清除浮动影响：一个盒子包含内容和一个浮动盒子，在没有设置BFC的情况下，盒子的高度由内容的高度决定，当给父容器设置overflow: auto;，父元素形成了BFC区域，区域的高度由内容和浮动子元素共同决定；
    2. 解决垂直方向上的外边距重叠问题；
    3. 自适应两列布局（float + overflow）

# Q3：自适应两列布局方案：

  1. float + margin
  2. calc()
  3. float + overflow:hidden
  4. flex布局

# Q4：如何理解层叠上下文？

  • 层叠上下文类似于BFC的一种布局区域，这个区域内主要涉及到元素的一个层叠顺序，即元素在页面上展示的层级划分；在这个区域中，DOM流后面的元素会覆盖前面的元素；
  • 基本层叠顺序：层叠上下文(background/border) < 负z-index元素 < block块状水平盒子 < float浮动盒子 < inline/inline-block水平盒子 < z-index:auto/0或者不依赖z-index确定顺序的层叠上下文 < 正z-index元素
  • 如何创建层叠上下文？
    1. 根元素html——根层叠上下文
    2. 定位元素和z-index不是auto的元素
    3. z-index不是auto的flex项(父元素是display:flex|inline-flex)
    4. 包含transform!=none/opacity!=1/fliter!=none/-webkit-overflow-scrolling:touch等css属性

# Q5：响应式布局的方案有哪些？

  1. 固定尺寸加媒体查询；
  2. 百分比布局；
  3. rem布局；

# 百分比布局相对于rem布局的优缺点？

  • 优点：用法简单，不需要js的计算；没有兼容性；
  • 缺点：计算麻烦，缺乏扩展性。

# rem的实现原理

  • 根据设计图的尺寸以及转化的比例，以及设备的尺寸计算出根节点html的字体大小，由于属性具有继承性，所以子元素的尺寸就可以根据根节点的大小进行计算，计算公式为换算比例/(设计稿的尺寸/设备尺寸)；

# Q6：如何理解弹性盒布局？

  • 弹性盒布局就是flex布局，通过设置display:flex;定义了一个弹性容器，里面包含了弹性子元素，弹性盒布局规定了弹性子元素在容器内的布局方式，而子元素内部和容器外部的元素不受此属性影响

# Q7：link和@import的区别？

  1. link是XHTML的标签，@import是CSS中style标签内引入外部css的API
  2. link是在html加载后就开始加载进入，@import是在页面加载完成之后才会加载；
  3. 用JS操作引入外部样式表只能通过link标签；
