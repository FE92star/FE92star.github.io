# JS基础知识

# Q1：script标签的属性

  • script是用来引入JS代码的html标签，包含以下属性：
    1. type：表示编写代码使用的脚本语言的内容类型，默认都是text/javascript；
    2. src：用于表示引入外部脚本的链接地址；
    3. defer和async：都是指引入的外部脚本会异步加载，防止阻塞html的解析，区别在于defer表示的是等待执行，即等整个页面加载结束之后再执行，虽然已经被异步加载；async则表示再不影响页面解析和其他脚本执行的情况下，立即下载该脚本代码；

# Q2：JS的数据类型

  • 包含基本类型和引用类型
  1. 基本类型：boolean,string,number,null,undefined,引用类型：object
  2. 区别在于：基本类型的变量赋值时完全赋值的，新的变量具有和赋值变量不同的内存空间，而引用类型在赋值的时候，是将对象的指针赋值给新变量，没有新开内存空间，因此指向的还是同一个堆内存中的对象，对新变量的操作会反应到赋值对象上面；
  3. 上面的现象的原因：基本类型的变量是存储在栈内存中，而引用类型的指针是存储在栈中，而指向的是存储在堆内存中的真正的对象；

# Q3：JS的执行机理（延申到作用域，作用域链，执行上下文）

  • JS是单线程语言，代码的执行顺序是从上到下依次执行，代码的执行是发生在执行环境栈（Ectack）中，可以类比成一个加工厂，所有的JS代码都必须经过工厂的加工处理，原理如下：
    1. JS代码进入执行栈加工厂中，首先初始化全局环境window，将全局环境压至栈底，并会生成一个全局变量对象，并在整个JS生命周期中一直存在，
    2. 执行函数A，A进入栈顶，并初始化A的执行上下文，执行上下文包含3个部分：变量对象（函数为活动对象），作用域链，this指向；变量对象包含：arguments对象，函数声明和变量声明； 执行结束之后，A的执行上下文被弹出，进入下一个函数的执行环节；
    3. 上述过程总结：作用域是指对变量的引用的权限
