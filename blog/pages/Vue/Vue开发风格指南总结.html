# Vue开发风格指南总结——规则归类

# 优先级A——必要的

  • 使用这些规则可以帮助规避很多错误
  1. 组建名可以使用多个单词进行命名，防止和HTML元素相冲突，类似todo-item这种命名方式
  2. data的值必须是一个返回对象的函数
  3. props定义需要更详细，包括默认值，类型，以及是否是必须属性
  4. v-for设置键值，提高渲染效率
  5. scoped为组件样式设置作用域

# 优先级B——强烈推荐

  • 应用这些规则能改善代码的可读性和开发体验
  1. 单个文件组件命名应该以大写字母开头，或者以-符号连接
  2. 同类的组件命名时可以加上特定共有的命名前缀，比如Base,Input等
  3. 父子组件——子组件的命名应该以父组件名作为前缀命名
  4. props在定义时遵循驼峰式命名规则，在使用时可以采取-连接符进行使用
  5. 多属性组件可以换行书写，保证可读性
