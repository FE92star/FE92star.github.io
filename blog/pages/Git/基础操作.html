# git基础操作总结

# 常规操作流程

  1. git status => git add .(放入缓存区) => git status => git commit -m "提交描述"（提交到本地仓库，即工作区） => git status => git pull（拉取线上的代码到本地仓库） => git status(查看没有问题和冲突的情况下再提交代码) => git push（提交本地代码到线上）
  2. git push -f orign 分支名——本地代码强制推到线上，保证本地代码是最新并且没有错误的情况下才可以操作，一般不建议这种操作

# 分支操作

  • 查看分支
    1. git branch -a（查看线上的分支）
    2. git branch （查看本地分支）
  • 分支新建
    1. git checkout -b "分支名"——在你当前所在的分支下面新建分支，新建的分支是和所在的分支的代码是保持同步的；
    2. 本地分支提交到远程——git push origin "分支名"；
    3. 拉取远程分支并切换到该分支——git fetch origin "远程分支名":"本地分支名"；
  • 分支合并
    1. git merge "分支名"——表示以当前分支作为基础，把对应分支合并到当前分支；一般用于本地开发分支测试完成需要上线，合并到master分支上；
    2. 合并之前确保本地分支没有问题，再切换到master分支，开始合并，解决冲突，提交至远端；
  • 分支删除
    1. 删除本地分支——git branch -D/d "本地分支名"；
    2. 删除远程分支——git push origin --delete "远程分支名"；

# 代码回滚

  • git reset
    1. git reset --merge commit-代表提交记录的编号——在被污染的工作区中回滚merge或者pull；
    2. git reset --merge ORIG_HEAD——代码已经提交到工作区中，并且已经git pull远程代码，发现pull的代码有问题的时候，就可以使用这个指令回滚到pull之前，并且提交到工作区的代码不会被清除；
    3. git reset --keep commit-提交编号——保留工作区并丢弃旧分支的一些提交
      // 假如正在编辑一些文件，并且已经提交，接着继续工作，
  // 然后发现当前工作内容应该属于另一个分支，与之前的提交无关，
  // 此时，可以开启另一个新分支，并且保留工作区的内容；
  git tag start
  git checkout -b branch1
  edit
  git commit
  edit
  git checkout -b branch2
  git reset --keep start
  // 1.把branch1的改变提交了；
  // 2.发现这个提交内容不属于branch1，此时新建分支branch2，并切换到branch2分支上；
  // 3.此时就可以把branch1上的提交记录清除掉，并保持工作区内容不变，即以改动的内容还在。

# special操作

  • 主要是关于解决bug的技巧（都需要git bash上面进行操作）

    1. 删除git🔒：rm -f .git/index.lock——git操作失败，可能就是由于git🔒的问题；
    2. 删除本地git仓库：rm .git -rf——本地仓库不删除，就无法新建git仓库；
    3. 删除git add .异常错误提示warning——git config --global core.autocrlf false。

  • 本地有多个git项目，用别名alias快捷切换git仓库

    1. 在C:\Users\Administrator目录下新建一个.bashrc可读文件，编辑这个文件（eg——alias licai='cd D:/\work/\wamp/\www/\51wealthy'，licai表示别名，后面是项目的绝对路径）。可以添加多个；
    2. 重启git，即可实现别名快捷切换。

  • git添加到环境变量

    1. 找到系统的高级设置里面的环境变量设置；
    2. 向环境变量里面添加一个新的path:'git在本机中安装，bin目录所在的绝对路径'，即可全局引用git操作；

# 基本应用

  • 本地代码上传至GitHub操作步骤：
      // github新建好仓库，在本地项目最所在位置进行操作；
  1. git init; //初始化git仓库
  2. git add .
  3. git commit -m "init project"
  4. git remote add origin "GitHub远程仓库地址" //连接远程仓库
  5. git push -u/f origin master //强推
